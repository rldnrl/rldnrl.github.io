다형성#

note

제네릭 타입
여러 장소에 타입 수준의 제한을 적용할 때 사용하는 플레이스홀더 타입. 다형성 타입 매개변수라고도 부른다.


type Filter = {    <T>(array: T[], f(item: T) => boolean): T[]}

이것의 의미는 다음과 같다.
"지금은 이 타입이 무슨 타입인지 알 수 없으니 누군가 filter를 호출할 때마다 타입스크립트가 타입을 멋지게 추론해주기를 바란다."

함수의 매개변수가 함수를 호출할 때 건네진 인수로 매번 다시 한정되듯이, T도 filter를 호출할 때마다 새로운 타입으로 한정된다.

제네릭은 함수의 기능을 더 일반화하여 설명할 수 있는 강력한 도구이다. 제네릭은 코드를 일반화하고, 재사용성을 높이고, 간결하게 유지하는 데 도움을 준다.

제네릭을 어디에 위치시키느냐에 따라서 타입이 한정되는 범위가 달리진다.

type Filter = {    <T>(array: T[], f: (item: T) => boolean): T[]}
const filter: Filter = (array, f) => {    // ...}
type Filter<T> = {    (array: T[], f: (item: T) => boolean): T[]}
const filter: Filter = (array, f) => { // error. 제네릭 타입은 한 개의 타입 인수를 요구    // ...}
const filter: Filter<number> = (array, f) => {    // ...}
type Map = {    <T, U>(array: T[], f: (item: T) => U): U[]}

note

제네릭 타입 추론
대부분의 상황에서 타입스크립트는 제네릭 타입을 잘 추론한다. 그러나 제네릭도 명시적으로 지정할 수 있다. 제네릭 타입을 명시할 때는 모든 필요한 제네릭 타입을 명시하거나 반대로 아무것도 명시해서는 안 된다.


function map<T, U>(array: T[], f(item: T) => U): U[] {    // ...}
map(('a', 'b', 'c'), _ => _ === 'a') // OK!map<string, boolean>(('a', 'b', 'c'), _ => _ === 'a') // OK!map<string>(('a', 'b', 'c'), _ => _ === 'a') // error. 두개의 타입인수가 필요map<string, boolean | string>(('a', 'b', 'c'), _ => _ === 'a') // OK!map<string, number>(('a', 'b', 'c'), _ => _ === 'a') // error.

타입스크립트는 제네릭 함수로 전달한 인수의 정보를 이용해 제네릭의 구체 타입을 추론하므로 때로는 다음과 같은 상황이 벌어질 수 있다.

const promise = new Promise(resolve => resolve(45))promise.then(result => result * 4) // error

타입스크립트는 result를 {}로 추론했다. 그것은 타입스크립에게 충분한 정보를 제공하지 않았기 때문이다. 타입스크립트는 제네릭 함수의 인수에만 의지해서 제네릭 타입을 추론하는데, 인수가 아무것도 없으면 기본적으로 T를 {}로 간주한다.

Promise의 제네릭 타입 매개변수를 명시해서 이 문제를 해결할 수 있다.

const promise = new Promise<number>(resolve => resolve(45))promise.then(result => result * 4) // error

tip

제네릭 타입 별칭 Array, ReadonlyArray 타입도 타입 별칭이다.


click이나 mousedown 같은 DOM 이벤트를 설명하는 MyEvent 타입을 정의하자.

type MyEvent<T> = {    target: T    type: string}

타입 별칭에서는 타입 별칭명과 할당 기호(=) 사이에만 제네릭 타입을 선언할 수 있다. target 프로퍼티는 <button />, <div /> 등 이벤트가 발생한 요소를 가리킨다. 예를 들어서 버튼 이벤트는 다음처럼 표현할 수 있다

type ButtonEvent = MyEvent<HTMLButtonElement>

MyEvent 같은 제네릭 타입을 사용할 때는 타입이 자동으로 추론되지 않으므로 타입 매개변수를 명시적으로 한정해야한다.

const myEvent: Event<HTMLButtonElement | null> = {    target: document.querySelector('#myButton'),    type: 'click'}

한정된 다형성#

  1. 일반 TreeNode
  2. 자식을 갖지 않는 TreeNode인 LeafNode
  3. 자식을 갖는 TreeNode인 InnerNode
type TreeNode = {    value: string}
type LeafNode = TreeNode & {    isLeaf: boolean}
type InnerNode = TreeNode & {    children: [TreeNode] | [TreeNode, TreeNode]}

mapNode 함수를 구현해보자.

요구 사항은 다음과 같다.

TreeNode의 서브타입을 인수 받아 같은 서브타입을 반환해야한다.

const a: TreeNode = { value: 'a' } // TreeNodeconst b: LeafNode=  { value: 'b', isLeaf: true } // LeafNodeconst c: InnerNode = { value: 'c', children: [b] } // InnerNode
const a1 = mapNode(a, _ => _.toUpperCase()) // TreeNodeconst b1 = mapNode(b, _ => _.toUpperCase()) // LeafNodeconst c1 = mapNode(c, _ => _.toUpperCase()) // InnerNode
function mapNode<T extends TreeNode>(node: T, f: (value: string) => string): T {    return {        ...node,        value: f(node.value)    }}

여러 제한을 적용한 한정된 다형성#

인터섹션으로 제한들을 이어 붙이면 된다.

type HasSides = { numberOfSides: number }type SidesHaveLength = { sideLength: number }
function logPerimeter<    Shape extends HasSides & SidesHaveLength>(s: Shape): Shape {    console.log(s.numberOfSides * s.sideLength)    return s}
type Square = HasSides & SidesHaveLengthconst square: Square = { numberOfSides: 4, sideLength: 3 }logPerimeter(square)

한정된 다형성으로 인수의 개수 정의하기#

가변 인수 함수(임의의 개수의 인수를 받는 함수)에서도 한정된 다형성을 사용할 수 있다.

function call(    f: (...args: unknown[]) => unknown,    ...args: unknown[]): unknown {    return f(...args)}
function fill(length: number, value: string): string[] {    return Array.from({ length }, () => value)}

표현하려는 제한

  • f는 T 타입의 인수를 몇개 받아서 R 타입을 반환하는 함수이다. 인수가 몇개 인지는 미리 알 수 없다.
  • call은 f 한 개와 T 몇 개를 인수로 받으며 인수로 받은 T들은 f가 다시 인수로 받는다. 마찬가지로 인수가 몇 개인지 미리 알 수 없다.
  • 은 f의 반환 타입과 같은 R 타입을 반환한다.
function call<T extends unknown[], R>(    f: (...args: T) => R,    ...args: T[]): R {    return f(...args)}
  1. call은 가변 인수 함수로 T와 R 두 개의 타입 매개변수를 받는다. T는 unknown[]의 서브타입, 즉 어떤 타입의 배열 또는 튜플이다.
  2. call의 첫 번째 인수는 함수 f이다. f 또한 가변 인수 함수로 args와 같은 타입의 인수를 받는다. 따라서 args의 타입이 무엇이든 f 인수의 타입도 같다.
  3. f 함수 외에도 call은 임의의 개수의 매개변수 ...args를 추가로 받는다. args는 나머지 매개변수, 즉 임의의 개수의 인수를 받을 수 있는 매개변수다. args의 타입은 T이며 T는 배열 타입이어야 하므로 타입스크립트는 args용으로 전달한 인수를 보고 T에 걸맞은 튜플 타입을 추론한다.
  4. call은 R 타입(f의 반환 타입으로 한정됨)의 값을 반환한다.

제네릭 타입 기본값#

함수 매개변수에 기본값을 설정할 수 있듯이 제네릭 타입 매개변수에도 기본 타입을 지정할 수 있다. 특정 요소의 타입을 알 수 없을 때, 제네릭 타입에 기본값을 설정할 수 있다.

type MyEvent<T = HTMLElement> = {    target: T    type: string}

T가 HTML 요소로 한정되도록 T의 경계를 추가하면 다음과 같다.

type MyEvent<T extends HTMLElement = HTMLElement> {    target: T    type: string}

기본 타입을 갖는 제네릭은 반드시 기본 타입을 갖지 않는 제네릭의 뒤에 위치해야한다.

// Badtype MyEvent<    Target extends HTMLElement = HTMLElement,    Type: string> = {    target: Target,    type: Type}
// Goodtype MyEvent<    Type: string,    Target extends HTMLElement = HTMLElement> = {    target: Target,    type: Type}
