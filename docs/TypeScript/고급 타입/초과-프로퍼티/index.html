4. 초과 프로퍼티 확인

초과 프로퍼티 확인#

타입스크립트가 한 객체 타입을 다른 객체 타입에 할당할 수 있는지 확인할 때도 타입 넓히기를 이용한다.

type Options = {  baseURL: string  cacheSize?: number  tier?: 'prod' | 'dev'}
class API {  constructor(private options: Options) { }}
new API({  baseURL: 'https://api.mysite.com',  tier: 'prod'})

옵션의 철자를 틀리면 어떻게 될까?

new API({  baseURL: 'https://api.mysite.com',  tierr: 'prod'})
// error. options에 trierr라는 프로퍼티가 존재하지 않음

타입스크립트가 이를 검출할 수 있었던 건 초과 프로퍼티 확인 기능 덕분이다. Fresh 객체 리터럴 타입 T를 다른 타입 U에 할당하려는 상황에서 T가 U에는 존재하지 않은 프로퍼티를 가지고 있다면 타입스크립트는 이를 에러로 처리한다.


tip

Fresh한 객체 리터럴 타입#

타입스크립트가 객체 리터럴로부터 추론한 타입을 가리킨다. 객체 리터럴이 타입 어서션을 사용하거나 변수로 할당되면 Fresh한 객체 리터럴 타입은 일반 객체 타입으로 넓혀지면서 Fresh는 사라진다.


new API({ // 1  baseURL: 'https://api.mysite.com',  badTier: 'prod'} as Options)
const badOptions = { // 2  baseURL: 'https://api.mysite.com',  badTier: 'prod'}
new API(badOptions)
  1. 유효하지 않은 옵션 객체를 Options 타입이라고 어서션했다. 타입스크립트는 더 이상 옵션 객체를 Fresh한 것으로 취급하지 않으므로 초과 프로퍼티 확인을 수행하지 않으며, 따라서 아무 에러도 발생시키지 않는다.
  2. 옵션 객체를 변수 badOptions에 할당했다. 타입스크립트는 이 객체를 더 이상 신선한 객체로 보지 않으므로 초과 프로퍼티 확인을 수행하지 않으며, 따라서 아무 에러도 발생시키지 않는다.
