3. 불변성

원본 데이터 구조를 변경하는 대신 그 데이터 구조의 복사본을 만들되 그 중 일부를 변경한다.

type Color = {  title: string  color: string  rating: number}
type RateColor = (color: Color, rating: number) => Color
const colorLawn: Color = {  title: "잔디",  color: "#00FF00",  rating: 0}
const rateColor: RateColor = (color, rating) => {  color.rating = rating  return color}
console.log(rateColor(colorLawn, 5).rating) // 5console.log(colorLawn.rating) // 5

함수의 인자는 실제 데이터에 대한 참조이다. rateColor 함수 안에서 color의 rating을 변경하면 원본 colorLawn 객체의 rating도 바뀐다.

원본 객체가 변경되지 않게 구현을 해보자

const rateColor: RateColor = (color, rating) => {  return Object.assign({}, color, { rating })}
console.log(rateColor(colorLawn, 5).rating) // 5console.log(colorLawn.rating) // 0

스프레드 연산자(...)를 활용해서 구현할 수 있다.

const rateColor: RateColor = (color, rating) => {  return {    ...color,    rating  }}
console.log(rateColor(colorLawn, 5).rating) // 5console.log(colorLawn.rating) // 0

배열의 push 함수는 불변성 함수가 아니다.

type Color = { title: string }
type AddColor = (title: string, colorList: Color[]) => Color[]
const colorList: Color[] = [  { title: '파란색' },  { title: '노란색' },  { title: '초록색' },]
const addColor: AddColor = (title, colorList) => {  colorList.push({ title })  return colorList}
console.log(addColor('주황색', colorList).length) // 4console.log(colorList.length) // 4

원래의 배열을 유지하려면 concat을 사용하면 된다.

const addColor: AddColor = (title, colorList) => colorList.concat({ title })
console.log(addColor('주황색', colorList).length) // 4console.log(colorList.length) // 3

역시 스프레드 연산저를 사용할 수 있다.

const addColor: AddColor = (title, colorList) => [...colorList, { title }]
