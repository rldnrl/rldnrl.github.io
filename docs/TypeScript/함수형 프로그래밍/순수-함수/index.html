4. 순수 함수

파라미터에 의해서만 반환값이 결정되는 함수를 의미한다. 순수 함수는 최소 하나 이상의 인수를 받고, 인자가 같으면 항상 같은 값이나, 함수를 반환한다. 순수 함수에는 부수 효과가 없다.

순수 함수의 규칙

  1. 순수 함수는 파라미터를 최소 하나 이상 받아야한다.
  2. 순수 함수는 값이나 다른 함수를 반환해야한다.
  3. 순수 함수는 인자나 함수 밖에 있는 다른 변수를 변경하거나, 입출력을 수행해서는 안 된다.
tip

부수 효과란?
전역 변수를 설정하거나 함수 내부나 애플리케이션에 있는 다른 상태를 변경하는 것을 말한다. 순수 함수는 인수를 변경 불가능한 데이터로 취급한다.


순수하지 않은 함수#

type Person = {  name: string  canRead: boolean  canWrite: boolean}
type SelfEducate = (person: Person) => Person
const frederick: Person = {  name: "Frederick Douglass",  canRead: false,  canWrite: false}
const selfEducate: SelfEducate = () => {  frederick.canRead = true  frederick.canWrite = true  return frederick}
selfEducate()console.log(frederick)

결과

{  "name": "Frederick Douglass",  "canRead": true,  "canWrite": true}

순수한 함수#

type Person = {  name: string  canRead: boolean  canWrite: boolean}
type SelfEducate = (person: Person) => Person
const frederick: Person = {  name: "Frederick Douglass",  canRead: false,  canWrite: false}
const selfEducate: SelfEducate = (person) => ({  ...person,  canRead: true,  canWrite: true})
console.log(selfEducate(frederick))console.log(frederick)

결과

{  "name": "Frederick Douglass",  "canRead": true,  "canWrite": true}{  "name": "Frederick Douglass",  "canRead": false,  "canWrite": false}

note

순수함수는 테스트하기 쉽다.
순수 함수는 자신의 환경 또는 '세계'를 변화시키지 않기 때문에 복잡한 테스트 준비 과정이나 정리 과정이 필요하지 않다. 순수 함수를 테스트할 때는 함수에 전달되는 인수만 제어하면 되며, 인자에 따른 결괏값을 예상할 수 있다.


DOM 순수하지 않은 함수#

const Header = (text: string) => {  let h1 = document.createElement('h1')  h1.innerText = text  document.body.appendChild(h1)}
Header("Header() caused side effects")

이 함수는 함수나 값을 반환하지 않으며, DOM을 변경하는 부수 효과를 발생시킨다.


React를 활용한 순수한 함수#

type Props = Readonly<{  title: string}>
const Header = ({ title }: Props) => <h1>{title}</h1>
