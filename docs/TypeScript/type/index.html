Type

여러 가지 타입#

any#

프로그래머와 타입스크립트 둘 다 알 수 없는 상황에서 기본 타입인 any로 가정한다.


unknown#

타입을 미리 알 수 없는 값이 있을 때는 any 대신, unknown을 사용하면 된다. unknown은 타입을 정제하기 전까지는 타입을 사용할 수 없도록 강제한다. 사용할 수 있는 연산자는 (==, ===, || , &&, ?, !)이다. 자바스크립트의 typeof, instanceof 연산자로 정제할 수 있다.

let a: unknown = 30 // unknownlet b = a === 30 // booleanlet c = a + 10 // errorif (typeof a === number) {    let d = a + 10}
  1. 타입스트립트가 무언가의 타입을 unknwon이라고 추론하는 경우는 없다. unknown 타입을 사용하고자 한다면 개발자가 명시적으로 설정해야한다.
  2. unknown 타입이 아닌 값과 unknown 타입인 값과 비교할 수 있다.
  3. unknwon 값이 특정 타입이라고 가정하고 해당 타입에서 지원하는 동작을 수행할 수 없다. 먼저 타입스크립트에게 해당 값이 특정 타입임을 증명해야한다.

boolean#

boolean 타입은 true, false 두 가지 값을 갖는다.

사용할 수 있는 연산자: 

==, ===, || , &&, ?, !
let a = true; // booleanlet b = false; // booleanconst c = true; // true. 타입 리터럴let d: boolean = true; // booleanlet e: true = true; // true. 타입 리터럴let f: false = true // error. false 타입을 true 타입에 할당할 수 없음.

tip

타입 리터럴
오직 하나의 값을 나타내는 타입


number#

number 타입은 모든 숫자(정수, 소수, 양수, 음수, Infinity, NaN 등)의 집합이다.

let a = 1234 // numbervar b = Infinity * 0.10 // numberconst c = 5678 // 5678let d = a < b // booleanlet e: number = 100 // numberlet f: 26.218 = 26.218 // 26.218let g: 26.218 = 10 // error. 26.218 타입에 할당할 수 없음

bigint#

number 보다 더 큰 수를 표현할 수 있다. number는 253까지의 정수를 표현할 수 있다. 숫자형과 관련된 연산자를 사용할 수 있다.

let a = 1234n // bigintconst b = 5678n // 5678nvar c = a + b // bigintlet d = a < 1235 // booleanlet e = 88.5n // error. bigint 리터럴은 반드시 정수let f: bigint = 100n // bigintlet g: 100n = 100n // 100nlet h: bigint = 100 // error. bigint에 할당할 수 없음

string#

let a = 'hello' // stringvar b = 'python' // stringconst c = '!' // '!'let d = a + ' ' + b + c // stringlet e: string = 'react' // stringlet f: 'vue' = 'vue' // 'vue'let g: 'vue' = 'angular' // error. 'angular'를 'vue'에 할당할 수 없음

symbol#

객체와 맵에서 문자열 키를 대신하는 용도로 사용한다. 심벌 키를 사용하면 사람들이 잘 알려진 키만 사용하도록 강제할 수 있으므로 키를 잘못 설정하는 실수를 방지한다.

let a = Symbol('a') // symbollet b: symbol = Symbol('b') // symbolvar c = a === b // booleanlet d = a + 'x' // error. '+' 연산을 symbol에 적용 불가const e = Symbol('e') // typeof econst f: unique symbol = Symbol('f') // typeof flet g: unique symbol = Symbol('g') // error. unique type은 반드시 constlet h = e === e // booleanlet i = e === f // error. unique symbol 타입은 서로 겹치는 일이 없으므로 반드시 false

객체 타입#

타입스크립트의 객체 타입은 객체의 형태를 정의한다. 객체 타입만으로는 간단한 객체({}로 만든 객체)와 복잡한 객체(new를 사용해 만든 객체)를 구분할 수 없다. 이는 자바스크립트가 구조 기반 타입을 갖도록 설계되었기 때문이다. 타입스크립트도 자바스크립트 스타일을 선호한다.


tip

구조 기반 타입화
구조 기반 타입화에서는 객체의 이름에 상관없이 객체가 어떤 프로퍼티를 갖고 있는지 따진다. 일부 언어에서는 덕 타이핑이라고 한다. (겉 표지만 보고 책을 판단하지 않는 것과 같은 원리다.)


tip

객체 리터럴 문법
"이런 형태의 물건이 있어!"라고 말하는 것이다.


let a: {    firstName: string    lastName: string} = {    firstName: 'david',    lastName: 'thomas'}

프로퍼티를 추가하거나 필요한 프로퍼티를 제공하지 않으면 에러가 발생한다.

let a: {b: number}a = {} // error. 타입에 필요한 'b'가 없음a = {    b: 1,    c: 5} // error. 객체 리터럴은 알려진 프로퍼티만 지정할 수 있음

그렇다면 어떤 프로퍼티는 선택형이고, 나중에 프로퍼티가 추가될 수 있다는 걸 알려줄 수 있을까?

let a: {    b: number    c?: string    [key: string]: boolean}
a = { b: 1 }a = { b: 1, c: undefined }a = { b: 1, c: 'python' }a = { b: 1, c: 'python' 'hello': true}

tip

인덱스 시그니쳐
[key: T]: U 같은 문법을 인덱스 시그니쳐라고 부르며 타입스크립트에 어떤 객체가 여러 키를 가질 수 있음을 알려준다. "이 객체에서 모든 T 타입의 키는 U 타입의 값을 갖는다"라고 해석할 수 있다. 인덱스 시그니쳐를 이용하면 명시적으로 장의한 키 외에 다양한 키를 객체에 안전하게 추가할 수 있다.

인덱스 시그니쳐에서 한 가지 반드시 지켜야할 규칙이 있다. 키(T)는 반드시 number나 string 타입에 할당할 수 있는 타입이어야한다.

키 이름은 원하는 이름을 가져다 바꿔도 된다. 즉, key가 아니어도 된다.


let airplaneSeatingAssignments: {    [seatNumber: string]: string} = {    '34D': 'David Thomas'    '34E': 'Mark'}

객체 타입을 정의할 때 readonly 키워드를 사용해 읽기 전용으로 정의할 수 있다. 즉, 정의한 필드에 초깃값을 할당한 다음에는 그 값을 바꿀 수 없다. 객체 프로퍼티에 const를 적용한 듯한 효과를 낸다.


배열#

타입스크립트는 초기화할 때 배열을 추론한다. 빈 배열을 선언하게 되면 any[]로 추측한다.

let a = ['red'] // string[]a.push('blue') // string[]a.push(10) // error. string 타입의 배열에 number 타입 인수를 할당할 수 없음
let b = [1] // number[]b.push(10) // number[]b.push('python') // error. number 타입 배열에 string 타입 인수를 할당할 수 없음
let c = [1, 'python'] // (number | string)[]c.push(5) // (number | string)[]c.push(true) // error.
let f = [] // any[]f.push(1) // number[]f.push('python') // string[]

읽기 전용을 지원한다.

type A = readonly string[]type B = Readonly<string[]>type C = ReadonlyArray<string>

튜플#

배열의 서브 타입이다. 튜플은 길이가 고정되어 있고, 각 인덱스의 타입이 알려진 배열의 일종이다. 다른 타입과 달리 튜플은 선언할 때 타입을 명시해야한다.

let a: number = [1]let b: [string, string, number] = ['David', 'Thomas', 1965]

선택형을 지원한다.

// 방향에 따라 다른 값을 갖는 기차 요금 배열let trainFares: [number, number?][] = [    [3.75],    [8.25, 7.70],    [10.50]]

또한 최소 길이를 갖도록 지정할 때는 나머지 요소(...)를 사용할 수 있다.

let friends: [string, ...string[]] = ['Sara', 'Tali', 'Chloe']let list: [number, boolean, ...string[]] = [1, false, 'a', 'b', 'c']

읽기 전용을 지원한다.

type A = readonly [number, string] // readonly [number, string]type B = Readonly<[number, string]> // readonly [number, string]

enum(열거형)#

해당 타입으로 사용할 수 있는 값을 열거하는 기법이다. 열거형은 키를 값에 할당하는 순서가 없는 자료구조이다. 키가 컴파일 타임에 고정된 객체라고 생각하면 쉽다. 따라서 타입스크립트는 키에 접근할 때 주어진 키가 실제 존재하는지 확인할 수 있다.


tip

열거형의 이름은 단수 명사로 쓰고, 첫 문자는 대문자로 하는 것이 관례이다. 키도 앞 글자를 대문자로 표시한다.


enum Language {    English,    Spanish,    Russian}

타입스크립트는 자동으로 열거형의 각 멤버에 적절한 숫자를 추론해 할당하지만, 명시적으로 설정할 수도 있다.

enum Language {    English = 0,    Spanish = 1,    Russian = 2}

보통 객체에서 값을 가져올 때처럼 점 또는 괄호 표기법으로 열거형 값에 접근할 수 있다.

let myFirstLanguage = Language.Englishlet mySecondLanguage = Language[Spanish]

타입스크립트는 여러 열거형 정의 중 한가지 값만 추론할 수 있으므로 열거형을 분할할 때 주의해야 하며, 다음 예처럼 각 열거형 멤버에 명시적으로 값을 할당하는 것이 좋다.

enum Language {    English = 0,    Spanish = 1}
enum Language {    Russian = 2}

계산된 값을 사용할 수도 있으므로 모든 값을 정의할 필요는 없다. (빠진 값은 타입스크립트가 추론한다.)

enum Language {    English = 100,    Spanish = 200 + 300,    Russian // 타입스크립트가 500 다음 숫자인 501로 추론}

열거형에 문자열 값을 사용하거나 문자열과 숫자값을 혼합할 수 있다.

enum Color {    Red = '#c10000',    Blue = '#007ac1',    Pink = 0xc10050,    White = 255}

더 안전한 열거형 타입인 const enum을 이용하면 타입스크립트가 안전하게 작업할 수 있다.

const enum Language {    English,    Spanish,    Russian}
// 유효한 enum 키 접근let a = Language.English // Color
// 유효하지 않은 enum 키 접근let b = Language.Tagalog // error. Tagalog 프로퍼티 없음
// 유효한 enum 키 접근let c = Language[0] // error. const enum은 문자열 리터럴로만 접근 가능
let d = Language[5] // error. const enum은 문자열 리터럴로만 접근 가능

const enum은 역방향 찾기를 지원하지 않으므로 열거형의 동작은 일반 자바스크립트 객체와 비슷하다. 또한 const enum은 기본적으로 아무 자바스크립트도 생성하지 않으며, 그 대신 필요한 곳에 열거형 멤버의 값을 채워 넣는다. 예를 들어 타입스크립트는 Language.Spanish가 사용된 모든 코드를 값 1로 바꾼다.

const enum Flippable {    Burger,    Chair,    Cup,    Skateboard,    Table}
function flip(f: Flippable) {    return 'flipped it'}
flip(Flippable.Chair) // 'flipped it'flip(Flippable.Cup) // 'flipped it'flip(12) // 'flipped it' (예상하지 못한 결과)

모든 숫자를 열거형에 할당할 수 있음을 알게 된다. 이 문제는 문자열 값을 갖는 열거형을 사용해 해결할 수 있다.

const enum Flippable {    Burger = 'Burger',    Chair = 'Chair',    Cup = 'Cup',    Skateboard = 'Skateboard',    Table = 'Table'}
function flip(f: Flippable) {    return 'flipped it'}
flip(Flippable.Chair) // 'flipped it'flip(Flippable.Cup) // 'flipped it'flip(12) // error. 12는 'Flippable' 매개변수 타입에 할당할 수 없음flip('Python') // error. 'Python'은 'Flippable' 매개변수 타입에 할당할 수 없음

결과적으로 숫자 값을 받는 열거형은 전체 열거형의 안정성을 해칠 수 있다.


tip

enum을 사용하는 것은 권장하는 타입은 아니다. enum을 사용하지 않고 const를 이용해보자


const Flippable = {    Burger: 'Burger',    Chair: 'Chair',    Cup: 'Cup',    Skateboard: 'Skateboard',    Table: 'Table'} as const
type FlippableValue = typeof Flippable[keyof typeof Flippable]
function flip(f: FlippableValue) {    return 'flipped it'}
flip("Burger") // 'flipped it'flip("Cup") // 'flipped it'

null, undefined, void, never#

undefined는 아직 값을 정의하지 않은 것이다.

null은 값이 없다는 것이다. 예를 들어 값을 계산하려 하면 에러가 발생한다.

void는 명시적으로 아무것도 반환하지 않는 함수의 반환 타입이다.

never는 절대 반환하지 않는 함수 타입을 가리킨다.

// number 또는 null을 반환하는 함수function a(x: number) {    if (x < 10) {        return x    }    return null}
// undefined를 반환하는 함수function b() {    return undefined}
// void를 반환하는 함수function c() {    let a = 2 + 2    let b = a * a}
// never를 반환하는 함수function d() {    throw new Error('error!')}
// never을 반환하는 또 다른 함수function e() {    while (true) {        doSomething()    }}

unknown이 모든 타입의 상위 타입이라면 never은 모든 타입의 서브 타입이다.
